PROJECT OVERVIEW - CAPITAL FARM BACKEND

Purpose
- Backend API for a farming game where players create accounts, manage identity and sessions, organize inventory, manage land, plant crops, harvest resources, and progress through an expanding production chain.

Current Tech Stack
- Python + FastAPI
- SQLAlchemy ORM + SQLite
- JWT auth (python-jose)
- Password hashing with passlib/bcrypt

What Is Already Implemented
- Player registration with username + email
- Player login with access token + refresh token flow
- Token refresh and logout with token/session revocation
- Protected routes via JWT validation
- Player profile module with metadata and stats container
- Wallet deposit endpoint and wallet history endpoint
- Structured inventory with item catalog, quantities, categories, and capacity limit
- Land grid bootstrap and land state management
- Crop type catalog
- Planting endpoint with seed consumption and plot occupancy validation
- Player crop listing endpoint

Core Domain So Far
- Player
  - id, username, email, email_verified, hashed_password, balance
- PlayerProfile
  - player_id, display_name, avatar_url, created_at
- PlayerStats
  - games_played, crops_planted, crops_harvested, total_earnings, total_expenses
- Inventory
  - player_id, capacity model, item container
- ItemCatalog
  - code, name, category
- InventoryItem
  - inventory_id, item_id, quantity
- WalletTransaction
  - player_id, transaction_type, amount, created_at
- LandPlot
  - player_id, x, y, soil_type, state, is_occupied
- CropType
  - code, name, growth_time_seconds, yield_quantity, base_value, seed_item_code
- PlayerCrop
  - player_id, crop_type_id, land_plot_id, planted_at, state

API Behavior (current)
- /register creates player, profile, stats, base inventory, and default land setup atomically
- /login returns bearer access token and refresh token
- /token/refresh rotates refresh session and returns new tokens
- /logout revokes active session and access token
- /me returns authenticated player account data
- /profile/me returns authenticated player profile and stats
- /wallet/deposit updates player balance
- /wallet/history returns player-scoped financial ledger
- /inventory/me returns structured inventory grouped by category
- /land/me returns the player land grid
- /crops/types returns available crop catalog
- /crops/plant validates land + seed and persists planting
- /crops/me returns player-owned crop records

Operational Notes for Codex
- Use venv at ./venv
- Start server with: python -m uvicorn app.main:app --reload
- Keep SECRET_KEY configured by environment when possible
- Prefer transaction-safe writes for player economy, inventory, land, and crop operations
